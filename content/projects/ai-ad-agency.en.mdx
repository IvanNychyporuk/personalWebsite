## A Company Run by AI Agents

<div style={{display:'grid',gridTemplateColumns:'1fr 1fr',gap:48,margin:'24px 0 48px'}}>
  <div>
    <p style={{fontSize:15,color:'#64748b',marginBottom:12}}>The central idea of this project is not just AI-assisted tooling — it is a complete company architecture where 12 specialized AI agents handle every department: executive strategy, compliance, engineering, QA, marketing, sales, and customer support.</p>
    <p style={{fontSize:15,color:'#64748b',marginBottom:12}}>Each agent has a defined role, a primary tool (Claude Cowork or Claude Code), a set of outputs it is responsible for, and a risk tier that determines how much autonomy it gets. Agents communicate via a typed message bus using a structured schema (<em>REQUEST → PLAN → APPROVE → EXECUTE → VERIFY</em>) so every decision is traceable.</p>
    <p style={{fontSize:15,color:'#64748b'}}>Human gatekeepers only appear at P4/P5 risk levels — for client-facing decisions, financial actions, or production deployments. Everything below that threshold runs autonomously.</p>
  </div>
  <div>
    <p style={{fontSize:15,color:'#64748b',marginBottom:12}}>When a client submits a campaign, the chain looks like this: <strong>A01 (CEO)</strong> sets the priority and publishes a REQUEST → <strong>A02 (Chief-of-Staff)</strong> writes the brief and runs market research → <strong>A08 (Marketing)</strong> generates the campaign concept, visual mockups, and storyboard → <strong>A05 (Product Manager)</strong> writes acceptance criteria → <strong>A10 (Builder)</strong> ships any code changes → <strong>A11 (QA)</strong> validates → <strong>A03 (Compliance)</strong> checks legal constraints before anything goes external.</p>
    <p style={{fontSize:15,color:'#64748b'}}>The same orchestration applies internally: feature requests go through Spec-to-Ship (A05 → A09 → A10 → A11), support tickets go through A06 triage, and cost/quality metrics are continuously monitored by the eval harness.</p>
  </div>
</div>
